## 한 줄 요약
ReFS v1.2·v3.3 내부 메타데이터 구조를 역분석하고, 메타데이터 기반·Write-at-Write 특성 기반 두 가지 삭제 데이터 복구 방법 제안.

## 핵심 발견
- **ReFS 메타데이터 구성**: Boot Record, Metadata block(v1.2: 48B 헤더 / v3.3: 80B 헤더), Super block(0x1E번 블록, Check point 위치 저장), Check point(primary/secondary 2개), Container Table, Object table(B+tree), Volume Label, UpCase, Allocator, Directory tree, SDS Data
- **삭제 동작 차이**: 휴지통 삭제 — Sequence number 변화, 파일 엔트리 플래그·오프셋 변경, LCN 정보 유지; 완전 삭제 — 메타데이터 플래그 변화, LCN 정보 일부 소거
- **복구 방법 1 — 메타데이터 기반**: Root Directory 내 삭제 엔트리 스캔 → Object ID로 데이터 위치 추적 → 파일 복원
- **복구 방법 2 — Write-at-Write**: ReFS 고유 특성(메타데이터 갱신 시 동일 위치 덮어쓰기 않고 새 위치에 기록) 활용 → 이전 메타데이터 블록에서 삭제 전 상태 복원
- 기존 도구(Encase v8.05, R-studio v5.4, Autopsy v4.9): ReFS 1.2까지만 지원, 복구 기능 미지원
