## 개요
클라우드·분산 시스템처럼 지속적으로 변화하는 환경에서 디지털 증거의 의미(meaning)를 보존하기 위한 참조 데이터(Reference Data) 수집·관리 전략.

## 메커니즘

**참조 데이터(Reference Data):**
- 트레이스를 분류·해석하기 위한 기준값
- 실험·비교를 통해 생성 (법과학 일반 원칙과 동일)
- 예: 특정 앱 버전의 정상 로그 패턴, 알려진 아티팩트 구조

**진화하는 시스템에서의 문제:**
- 앱/서버 업데이트 → 아티팩트 구조·의미 변경
- 수집 당시 참조 데이터와 트레이스의 버전 불일치 → 잘못된 해석
- 클라우드 서비스: 서버 코드가 수시로 배포되어 로그 형식 변경
- 모바일 앱: 잦은 업데이트로 아티팩트 위치·포맷 변경

**트레이스-참조 데이터 거리 모델:**
- X축: 참조 데이터 최신성 (생성 시점)
- Y축: 사용 가능성 (신뢰도)
- 거리가 멀수록 불확실성 증가 → 증거 가치 저하

## 탐지 방법 (Investigator)

**참조 데이터 관리 원칙:**
1. 트레이스 수집과 동시에 참조 데이터 수집
2. 참조 데이터에 버전 정보·수집 시각 태그
3. 앱/시스템 업데이트 주기에 맞춘 정기적 갱신
4. 참조 데이터 아카이브 유지 (과거 버전 포함)

**실무 적용:**
- 모바일 포렌식: 앱 최신 버전으로 별도 기기에서 동일 행위 재현 → 참조 데이터 생성
- 클라우드 포렌식: API 응답 구조 문서화, 타임스탬프 의미 확인
- 분산 시스템: 노드별 시계 동기화 상태 기록

## 한계 & 주의사항
- 참조 데이터 수집 자체가 자원 집약적
- 완전한 참조 데이터 확보가 불가능한 경우 (폐쇄형 서비스)
- 참조 데이터가 트레이스와 완전히 동일한 환경에서 수집되어야 유효
