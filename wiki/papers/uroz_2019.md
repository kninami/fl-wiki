## 한 줄 요약
Windows ASEP 4범주 분류체계를 구축하고, 메모리 포렌식으로 탐지 가능한 ASEP과 불가능한 ASEP을 실험으로 구분.

## 핵심 발견
- **ASEP 4범주**: System persistence mechanisms / Program loader abuse / Application abuse / System behavior abuse
- **레지스트리 기반 ASEP은 모두 메모리 포렌식으로 탐지 가능** (설정 범위 무관)
- 파일시스템 기반 ASEP(Startup folder, Shortcut manipulation, Scheduled tasks)은 파일 카빙 없이 메모리 포렌식만으로 완전 탐지 불가
- 실행 중인 프로세스 분석 필요: Trojanized binaries, DLL hijacking, Office add-ins
- ASEP 실행 순서 실험적 확인: Winlogon → Shell → RunOnceEx → Active Setup → Startup folder → Run keys
- **Winesap** Volatility 플러그인 개발 (GNU GPLv3, gitlab.unizar.es/rrodigu/winesap)
