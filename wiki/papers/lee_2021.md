## 한 줄 요약
ReFS 파일시스템의 저널링 파일(Logfile, Change Journal) 구조를 커널 역공학으로 규명하고, 포렌식 방법론 및 분석 도구(ARIN) 개발.

## 핵심 발견
- **ReFS Logfile**: redo 연산만 기록 (NTFS $LogFile은 redo+undo); 완전히 새로운 구조(MLog 시그니처)
- **Change Journal**: $USNjrnl과 유사하지만 기본 비활성화; USN_RECORD_V3 포맷 사용 (128bytes, File Reference Number 32bit)
- ReFS는 AOW(Allocate On Write) 트랜잭션 모델 → undo 연산 불필요
- Logfile Entry: Entry Header + Log Record Header + Redo Records 구성
- Redo Record: 28종 파일시스템 연산 opcodes 포함
- ARIN(Awesome ReFS Investigation Tool): 오픈소스, Change Journal·Logfile 분석
- ReFS 메타데이터 파일: Object ID Table, Container Table, Parent Child Table, File System Metadata, Logfile Information Table
