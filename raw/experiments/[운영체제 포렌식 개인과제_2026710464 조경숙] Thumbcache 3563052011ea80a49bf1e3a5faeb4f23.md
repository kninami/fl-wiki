# [운영체제 포렌식 개인과제_2026710464 조경숙] Thumbcache

# Thumbcache 란

#### 정의

Windows OS에서 사용되는 폴더 내의 이미지, 기타 파일 등의 대표 이미지를 보여주는 썸네일 데이터. Thumbcache.db 파일에 썸네일 이미지를 저장하며, 원본 파일명, 날짜 및 시간 정보 등을 포함. Windows Version 에 따라 Store의 이름과 위치에 차이가 존재[1]. 

![image.png](%5B%EC%9A%B4%EC%98%81%EC%B2%B4%EC%A0%9C%20%ED%8F%AC%EB%A0%8C%EC%8B%9D%20%EA%B0%9C%EC%9D%B8%EA%B3%BC%EC%A0%9C_2026710464%20%EC%A1%B0%EA%B2%BD%EC%88%99%5D%20Thumbcache/image.png)

- `Thumbcache_32, 96, 256, 1024` : 이미지 썸네일의 크기(해상도)에 따라 제각기 다르게 저장됨 (작은 아이콘/보통 아이콘/큰 아이콘/사진 뷰어-1024x1024px)
    - Windows11 기준으로, Thumbcache_16/32/48/96/256/768/1280/1920/2560.db 존재
- `Thumbcache_idx.db` : 다른 Thumbcache DB의 메타데이터 보유

![image.png](%5B%EC%9A%B4%EC%98%81%EC%B2%B4%EC%A0%9C%20%ED%8F%AC%EB%A0%8C%EC%8B%9D%20%EA%B0%9C%EC%9D%B8%EA%B3%BC%EC%A0%9C_2026710464%20%EC%A1%B0%EA%B2%BD%EC%88%99%5D%20Thumbcache/image%201.png)

| **Title** | **Description** |
| --- | --- |
| Entry Offset | DB 파일에서 시작되는 위치 |
| Entry Size | 전체 크기 (Header + Data 포함) |
| Data Offset | Entry 내부에서 이미지 데이터가 시작되는 위치 |
| Data Size | 실제 썸네일 이미지 데이터의 사이즈 |
| Data Checksum | 이미지 데이터 무결성 검증 값 |
| Header Checksum | Entry Header 무결성 검증값 (메타데이터) |
| Cache Entry Hash | 원본 파일을 식별하기 위한 해시값 |

#### 법정 효력

사건에 따라 차이가 있지만, 아동성착취물 사건에 있어 관련 썸네일 캐시가 존재하는 경우, 다른 정황들과 결합하여 증거로 인정되는 경우 존재[1]. 

- 영국 Attorney General v Roberts 사건([2010] JRC088)
    - 아동 성착취물 원본 파일은 존재하지 않았으나, 관련 썸네일 캐시가 존재
    - 브라우저 로그 등에서 아동 성착취물로 의심되는 제목의 영상 파일 다운로드 및 검색 내역 존재 &일부 스트리밍 흔적 있음
    - 이를 통해 종합적으로 아동성착취물을 시청 및 소지했다는 증거로 인정됨

단, 썸네일 캐시가 존재한다고해서 해당 파일을 ‘보았다’고 증명할 수는 없음. 파일탐색기로 해당 폴더를 열었을 때, 해당 이미지를 직접 보지 않아도 썸네일 캐시가 생성됨.

# Thumbcache 의 최대 크기

### 실험 방식

- 약 6600여개 이미지 파일 준비하여 ‘큰아이콘’, ‘보통아이콘’, ‘매우큰아이콘’, ‘미리보기’로 파일 열기 수행
    - 로컬 드라이브 2000여개, 외장하드 3300여개 준비
- `C:\Users\name\AppData\local\Microsoft\Windows\Explorer` 하위 Thumbcache DB 용량 확인
- Thumbcache viewer 로 DB 열어 실제 썸네일 이미지가 가시적으로 보이는지 여부 확인

| **뷰 모드** | **저장되는 DB (추정)** |
| --- | --- |
| 보통 아이콘 | _96 |
| 큰 아이콘 | _96 ~ _256 |
| 아주 큰 아이콘 | _256 |
| 미리보기 패널 | _256~1280 |
| 미리보기 패널(가장 크게) | _1280~_2560 |

### 실험 결과

- 최소값 1KB (header + 기본적인 DB 구조만 있는 상태)
- 최대값 (용량 | 엔트리)
    - thumbcache_16/32: 1,024kb (1MB) | 19개
    - thumbcache_48 : 2,048kb (2MB) | 272개
    - thumbcache_96 : 96,256kb (96.2MB) | 3750개
    - thumbcache_256 : 66,560KB (66.5MB) | 6679개
    - thumbcache_1280 : 65,536KB (65.5MB) | 360개
    - thumbcache_2560 : 1,637,376KB (1.63GB) | 3146개

### 특이사항

- 엔트리 개수는 달라지기는 하지만, _2560은 3200개를 넘지 않음.
- 이미지의 사이즈와 미리보기 패널 사이즈가 동일해도 (3000x2000, 300dpi), 이미지에 따라 _1280과 _2560에 저장되는 파일이 다름
    - 주로 모바일 기기로 촬영한 사진의 경우 동일한 해상도를 가져도 _2560에 저장
    - 전문적인 카메라 장비로 촬영한 사진의 경우 _1280에 저장

> 
> 
> 
> 원인: EXIF 내장 썸네일
> 전문 카메라 (→ _1280) RAW 파일이나 카메라 JPG는 촬영 시 EXIF 안에 미리 렌더링된 썸네일이 내장됩니다. 카메라 제조사가 보통 1280px 수준의 썸네일을 내장하기 때문에 Windows가 그걸 그대로 꺼내서 _1280에 저장합니다.
> 모바일 폰 (→ _2560) 모바일 JPG는 EXIF 내장 썸네일이 없거나 매우 작습니다. Windows가 원본 이미지를 직접 디코딩해서 썸네일을 생성하기 때문에 더 큰 _2560에 저장됩니다.
> 
> *Claude Sonnet 4.6*

### 케이스2. 최대용량 상태에서 신규 이미지 열람 (엔트리 덮어씌우기)

- _2560이 9개 이미지를 추가적으로 열었을 때, 6개 이미지가 삭제됨.
    - 6개 이미지의 용량 (~647KB) > 9개 이미지의 용량 (~369KB)
    - LRU(Least Recently Used, 가장 오랫동안 사용되지 않은 데이터를 캐시 메모리에서 우선적으로 제거) 알고리즘에 따라 삭제한다고 알려져있지만, FIFO(First In First Out) 방식일 가능성도 있음.
